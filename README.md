# operating-system-projects
List of projects done during Operating System coursework. 

# Project - 1
In this project, we design and implement a multi process and multi thread matrix multiplication program by converting a given single-process matrix multiplication in C language. It aims to compare execution times of matrix multiplication using three approaches: serial processing, multi-processing, and
multi-threading by analysing how the application performance under different matrix dimensions. We also perform crash recovery in case of crashes to bring
back the system to a safe state where the system can operate normally.

# Project - 2
In this assignment, we try to find potential concurrency bugs such as race condition, deadlock and livelock and try to find a solution to solve this bug in four different applications namely – worker, stringbuffer, pbzip2 and aget. We further try to investigate, reproduce and resolve this bug in these applications.

# Project - 3
In this project, we implement a Virtual Memory Simulator to understand page tables and page replacement algorithms. It will simulate a 16-bit virtual machine
that supports a linear page table structure with up to 256 pages, 256 frames, and 8 processes. This is a simulator of virtual-to-physical address translation that, will handle page faults, free frame, and dirty frame lists in both physical memory and the swap disk. When all the frames are full, page replacement will be done with algorithms such as: FIFO, LRU, CLOCK, and CLOCK-PRO. 

# Project - 4
In this project, we implement a simple EXT-like file system simulator in order to understand the hierarchical directory and inode structures. We will be able to (1) browse the disk information, file and directory list, (2) create and delete files and directories, (3) read files. The functionality of the file system is similar to EXT file systems, but it does not include per-process open file table, permission and user management.
